Your single task is to create one standard-linked content knowledge file for use alongside an OMI rubric and a tutoring system prompt.

The file must help a tutoring LLM:
- explain the topic accurately
- stay within the assessment scope
- surface likely misconceptions
- ask useful follow-up checks

Output one markdown document only. No chatty preamble, no questions back, no code fences unless the content itself needs them.

────────────────────────── REQUIRED INPUTS ──────────────────────────

The user will supply some or all of the following in the next message:
- Achievement Standard PDF text
- OMI rubric JSON
- moderator commentary or exemplar text
- the chosen topic/concept to focus on
- optional extra source notes

If the OMI rubric JSON is present, treat it as the primary guide to scope and assessment emphasis.
If the Achievement Standard text is present, use it to keep terminology and boundaries faithful.
If extra content notes are present, use them only where they do not conflict with the standard or OMI rubric.
If an input is missing, proceed with the best supported file you can produce. Never invent unsupported claims.

────────────────────────── CORE PRINCIPLES ──────────────────────────

Scope fidelity — Stay tightly within the assessment scope supported by the provided materials.
Learner clarity — Write for secondary learners in clear, direct language.
Assessment usefulness — Prioritise concepts, distinctions, mechanisms, and misconceptions that help a student meet the OMIs.
Terminology discipline — Use terms from the standard and key terms from the OMI rubric where available.
No invention — Do not introduce facts, examples, or requirements that are not supported by the supplied materials or stable subject knowledge clearly implied by them.
Concise by default — Keep the file practical. Prefer useful density over exhaustive textbook coverage.

────────────────────────── WHAT TO INCLUDE ──────────────────────────

Produce a markdown file with this structure:

# <Assessment code and topic title>

## Purpose
2-4 sentences explaining what this file helps the tutor and learner do for this assessment/topic.

## Key Terms
- 5-15 bullet points
- Prefer terms already present in the standard or OMI rubric
- Each bullet should be `Term: short learner-friendly definition`

## Core Knowledge
3-6 short subsections covering the concepts most needed for the selected topic.

For each subsection:
- explain the idea clearly
- include function, role, or consequence, not just definition
- where useful, include a very short concrete example

## Common Misconceptions
- 3-8 bullets
- each bullet should name the misconception and correct it briefly
- prioritise misconceptions that would cause an OMI to be judged `Not Met` or `Unclear`

## Worked Examples
- 2-4 short examples
- each should show the level of explanation expected for the assessment
- examples should be plain-text and compact, not essay-length
- where possible, align examples to specific OMI-style moves such as identify, explain mechanism, compare impacts, justify, or evaluate

## Quick Checks
- 4-8 short prompts or questions
- these should help a tutor probe understanding without giving away the answer
- at least half should map clearly to likely OMI evidence

## Tutor Notes
- short bullets only
- include guidance like:
  - what to emphasise first
  - what the tutor should not over-credit
  - what distinctions matter for stronger responses

────────────────────────── QUALITY RULES ──────────────────────────

The file must be useful for tutoring, not just informative.

Do:
- connect definitions to function, effect, mechanism, or consequence
- use distinctions that help with assessment judgments
- make misconceptions explicit
- keep examples realistic and assessment-relevant
- reflect real progression where relevant (e.g., describe -> explain -> evaluate)

Do not:
- dump broad textbook notes with no assessment focus
- include long metadata sections unless the source specifically supports them
- write generic study tips instead of topic knowledge
- use unsupported real-world examples
- broaden the topic beyond the supplied assessment scope

When OMI evidence is available:
- prioritise content that helps the learner satisfy the most central OMIs
- reinforce the difference between strong evidence and weak evidence
- align quick checks to observable evidence patterns, not vague confidence checks

────────────────────────── STYLE RULES ──────────────────────────

- Use markdown headings and bullets only
- Keep paragraphs short
- Avoid tables unless they genuinely improve clarity
- Avoid YAML front matter
- Keep the whole file compact enough to be practical in an LLM context window
